Core Agent Skills

Two highly cohesive modules designed for LLM execution.

AML Rule Generator

Reads global crypto regulatory documents (FATF, MAS, VARA) from the references directory and synthesizes highly localized, machine-readable rules.json enforcement policies.

  • Localized JSON Rule Schemas
  • Zero-Tolerance Enforcement Configuration
  • Regulatory Redline Mapping

Address Screening (KYA)

Deep-scans a blockchain address up to 5 hops, mathematically maps fund flows, and aggressively evaluates exposure against your local rules.json to produce professional Markdown Audit reports.

  • Sub-second API Graph Navigation
  • Automated Evidence Tracing (A --$--> B)
  • Natural Language LLM Audit Reporting

Plug & Play Installation

Deploy effortlessly to OpenClaw, Claude Code, or any File-System LLM framework.

1. Clone into your Workspace

Pull the skills directly into your agent's environment.

git clone https://github.com/amlclaw/amlclaw.git ./skills/amlclaw

2. Install Dependencies

Python 3.10+ is required for the data extraction modules.

pip install -r aml-address-screening/requirements.txt

3. Environment Setup

Provide your Graph Provider API Key to allow deep chain tracing.

# Create .env file based on the example
cp .env.example .env

Example Agent Prompt

"Use the AML Address Screening skill to investigate Tron address TGE94... using our new rules.json file. Check both inflows and outflows up to 3 hops. Then give me a final executive summary."
