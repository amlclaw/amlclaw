# 中东加密企业反洗钱（AML）合规优化配置指南

本指南专为阿联酋迪拜（VARA监管）及中东地区（ADGM/SCA监管）的加密企业（特别是OTC大宗交易商）定制，严格依据阿联酋联邦法规及VARA监管框架，结合最新的AML风险标签分类进行了深度细化。

## 1. 基础合规阈值与系统全局设定
在中东地区经营，系统底层参数必须严格锚定法定红线，确保绝对合规：
*   **转账规则（Travel Rule）与 CDD 阈值**：**AED 3,500（约 USD 950）**。所有大于该阈值的交易均需强制执行Travel Rule并进行客户尽职调查（CDD）。
*   **记录留存期限（Record Keeping）**：**8 年**。系统需支持长达8年的数据长期回溯要求。
*   **资金来源追溯（Inflow Depth）**：**5 层（5 Hops）**。需穿透混淆层，有效识别经过多次跳转试图清洗的资金分层（Layering）。
*   **资金去向监控（Outflow Depth）**：**2-3 层**。重点监控直接接收方及次级接收方，防止资金经由跳板流向恐怖融资等高危地址。

## 2. 基于 AML 标签的细化风控规则配置
结合反洗钱(AML)风险标签分类指南，建议将风控规则分为以下响应级别：

### 2.1 🔴 严重违规类（风险等级：严重 1）—— 零容忍与直接冻结
对于涉及制裁、恐怖主义融资及严重金融犯罪的标签，系统风险评分须设为 100，实行零容忍，一旦命中立即冻结或拦截。

| 风险标签分类 (一级) | 具体 AML 标签包含 (二级) | 监管依据与处置动作 |
| :--- | :--- | :--- |
| **制裁 (Sanctions)** | 制裁实体 (Sanctioned Entity)<br>制裁管辖区 (Sanctioned Jurisdiction)<br>发出限制的国家/地址 (Prohibited Entity) | **处置**：出入金实时拦截、冻结资金。<br>**依据**：必须筛查 UN Consolidated List 及阿联酋本地恐怖分子名单 (UAE EOCN)。 |
| **恐怖主义融资 (Terrorism Financing)** | 恐怖组织 (Terrorist Organization) | **处置**：直接拦截，立即报告。 |
| **其他金融犯罪 (Other Financial Crimes)** | 涉嫌洗钱 (Suspected Money Laundering) | **处置**：直接拦截并冻结资金。 |
| **执法冻结 (Public Freezing Action)** | 执法机构冻结 (Law Enforcement Freeze/Seizure) | **处置**：零容忍拦截。 |
| **非法市场 (Illicit Markets)**| 人口贩卖 (Human Trafficking)<br>非法服务 (Illegal Services)<br>儿童虐待 (Child Sexual Abuse Material) | **处置**：直接拒绝入金并触发严重警报。 |

### 2.2 🚫 VARA 明令禁止类（风险等级：高风险 2）—— 阻断混淆与隐私资产
VARA 对“匿名增强型加密货币”及匿名性活动的容忍度为0%，相关活动属于严令禁止范畴。

| 风险标签分类 (一级) | 具体 AML 标签包含 (二级) | 监管依据与处置动作 |
| :--- | :--- | :--- |
| **混淆/隐私 (Obfuscation)** | 混币器 (Mixers)<br>隐私钱包 (Privacy Wallet)<br>隐私代币 (Privacy Token)<br>中心化跨链桥服务 (Centralized Bridge)<br>加密货币ATM (Crypto ATM/Kiosk) | **处置**：风险评分 100，直接拒绝入金。<br>**依据**：VARA 禁止接收匿名币及混币器（如 Tornado Cash）资金，视为最高风险。 |

### 2.3 🟠 网络犯罪类（风险等级：高/中风险 2）—— 动态阈值拦截
对于黑客入侵或链上欺诈类资金，需配置双重阈值进行拦截，以排除小额粉尘攻击（Dust Attacks）。

| 风险标签分类 (一级) | 具体 AML 标签包含 (二级) | 监管依据与处置动作 |
| :--- | :--- | :--- |
| **网络犯罪 (Cybercrime)** | 被盗资金 (Hacker/Thief)、勒索软件 (Ransomware)、网络钓鱼 (Phishing)、庞氏骗局 (Ponzi)、杀猪盘 (Pig Butchering)、项目方跑路 (Rug Pull)、蜜罐合约 (Honeypot)、虚假预售 (Fake Presale)、黑/灰产交易 (Black/Grey Industry) | **触发阈值**：Exposure > 10% 且 Amount > USD 100。<br>**处置**：触发 High 级别警报，识别暗网或欺诈资金，由 MLRO 介入。 |

### 2.4 🟡 敏感与高危实体（风险等级：高风险 3 / 低风险 6）—— 触发 EDD 与所有权验证
此类标签活动需触发增强尽职调查（EDD）及严格验证要求。

| 风险标签分类 (一级) | 具体 AML 标签包含 (二级) | 监管依据与处置动作 |
| :--- | :--- | :--- |
| **博彩 (Gambling)** | 无牌照博彩 (Unlicensed Gambling) | **处置**：阿联酋禁赌，触发 EDD，强制询问资金来源 (Source of Wealth)。 |
| **高风险实体 (High-Risk Entities)**| 高风险地区交易所 (CEX in High-Risk Jurisdiction)、被制裁交易所 (Sanctioned CEX)、弱管控交易所 (CEX with Weak AML/CFT Controls)、场外交易平台 (OTC Desk) | **处置**：严禁与高风险管辖区交互，若交互则触发 SEVERE 级别拦截。 |
| **辅助类 (Other Entities) - 非托管钱包**| 自托管钱包 (Self-custody Wallet)、AA钱包 (AA Wallet) | **处置**：标记为 High Risk。提币金额 > USD 950 时，强制要求所有权证明（Satoshi Test）。 |

## 3. 动态交易模式与交互规则 (Interaction Rules)
针对 OTC 业务中常见的规避监管行为，需要结合系统规则进行动态聚类监控：

*   **拆分交易监控 (Structuring/Smurfing)**：同一地址（或同一聚类簇）在 **24 小时内** 累计交易金额 > USD 950。即便单笔金额小于 $950，只要累计超过该阈值，系统必须自动合并计算并强制触发 CDD 要求与 Travel Rule 门槛。
*   **高风险 VASP 交互**：识别与高风险辖区（High Risk Jurisdictions，如FATF黑名单国家）交易所地址的交互，予以直接拦截（SEVERE）；同时建议将持牌合规交易所（如 Binance, Kraken）加入白名单以减少误报。

## 4. 事后监控与持续审查机制 (Ongoing Monitoring)
为满足 VARA 极高的情报响应要求与持续监控规定，需开启Ongoing Monitoring模块并配置严密的审查流程：

1.  **标签变动实时预警**：对 OTC 高净值客户进行**每日扫描（Daily）**，对零售客户在交易时进行实时扫描并**每周回溯**。若历史交易地址被新增标记为“被盗资金 (Hacker)”或“欺诈 (Scam)”等，系统需立即报警。
2.  **制裁名单时效响应**：当 UN 或阿联酋本地名单（UAE EOCN）更新时，系统必须在 **24 小时内** 完成所有存量地址的回溯扫描。
3.  **可疑交易报告（STR）与响应时效**：一旦由反洗钱报告官（MLRO）确认可疑交易，必须立即通过 **GoAML** 平台向阿联酋金融情报中心（UAE FIU）提交报告。若 FIU 提出进一步信息请求，企业必须在 **48 小时内** 作出回复。
